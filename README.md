# Sudoku Web Application — Frontend

The frontend interface for the Sudoku Web Application. Built with Vanilla JavaScript, it focuses on high performance, modern design aesthetics, and a flawless user experience.

## Features

- **Dynamic UI:** Clean, intuitive interface optimized for both desktop and mobile viewing with responsive CSS.
- **Real-time Validation & Feedback:** Visually indicates incorrect moves and highlights related cells instantly.
- **Smart Auto-Save:** Automatically saves your progress using `localStorage` so you never lose a game.
- **Glassmorphism Modals:** Modern pause and prompt dialogs with dynamic background blurring.
- **Keyboard Shortcuts:** Full keyboard support for navigation (Arrows), input (1-9), deletion (Backspace/Delete), undo (Ctrl+Z), and notes mode (N).

## 🚀 Architecture & Performance Optimizations

This project demonstrates advanced Vanilla JavaScript techniques to achieve framework-like reactivity and performance without the overhead of external libraries:

- **DOM Rendering Optimization:** Utilizes `DocumentFragment` to batch DOM insertions, completely eliminating Layout Thrashing during board generation (preventing 81 consecutive reflows).
- **O(1) Memory Caching:** Implements a 2D Array cache for all DOM nodes (`cellsCache`), bypassing expensive `document.querySelectorAll()` calls during high-frequency events like cell highlighting or typing.
- **Smart State Management:** Integrates a custom `Proxy` object alongside a custom **Debounce** function to automatically sync the game state to `localStorage` (Auto-save) without saturating disk I/O.
- **Vite Bundler:** Uses Vite for lightning-fast Hot Module Replacement (HMR) during development and highly optimized minification for production.

## Backend Integration

This frontend is designed to consume a RESTful API (e.g., a FastAPI backend) for heavy operations like puzzle generation and global validation. 
Ensure your backend is running, or adjust the endpoint URL in `src/js/services/api.js` to point to your production server.

## Installation & Setup

### Prerequisites
- Node.js (v18 or higher)
- npm (Node Package Manager)

### Development Server

1. Navigate to the frontend directory:
   ```bash
   cd frontend
   ```
2. Install dependencies:
   ```bash
   npm install
   ```
3. Start the Vite development server:
   ```bash
   npm run dev
   ```
   *The app will run at `http://localhost:5173` with Hot Module Replacement (HMR).*

### Building for Production

To create a minified, production-ready build:
```bash
npm run build
```
This generates a `dist/` folder containing the highly optimized static assets. You can preview the production build locally using:
```bash
npm run preview
```

## Usage

1. Select a difficulty level and click **New Game**.
2. Click on empty cells and use the provided numpad or your physical keyboard to input numbers.
3. Toggle "Notes" mode (or press `N`) to pencil in possible numbers.
4. If you accidentally close the tab, the app's Auto-Save will prompt you to restore your progress when you return.

## Deployment

Hosted on **GitHub Pages**. The repository is configured to automatically deploy the `dist/` folder generated by Vite.
